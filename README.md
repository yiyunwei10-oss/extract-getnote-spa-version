# Get笔记文案提取工具 (SPA 深度提取版)



&#x20;     一个基于 Playwright 的高效自动化工具，用于提取 Get笔记知识库中的所有文章，保存为 Markdown 文件。



## 🙏 致谢与声明

&#x20;     本项目的大部分基础架构衍生自 \\\\\\\\\\\\\\\[dontbesilent2025/extract-getnote-articles](https://github.com/dontbesilent2025/extract-getnote-articles)。

&#x20;     由于原目标网站改版为单页应用 (SPA) 架构，导致原版工具的并发提取失效。本项目 (`v3.0.0`) 对核心抓取逻辑进行了 100% 的重构，采用智能单线程跳转模式，完美适配最新版 Get笔记。特此向原作者致谢！



## ✨ 新版功能特点

* 🤖 **智能单页跳转捕获**：完美解决 SPA 架构下 URL 不变导致的抓取失效问题。
* 🎯 **深度长文遍历**：不再局限于提取 AI 摘要，自动穿透“原链接”阻断，智能遍历抓取完整正文段落。
* ⏱️ **拟人化安全提取**：内置 3 分钟的宽裕扫码登录等待时间，单线程平滑后退，极大降低封号和被拦截的风险。
* 📝 **格式化输出**：保存为结构化的 Markdown 文件，包含标题、原链接和完整内容，自动过滤非法字符。
* 📁 **智能命名**：使用博主名称自动命名文件夹，完美处理重名冲突。
* 💾 **断点续传**：自动跳过已提取的文章，支持随时中断和继续

## 安装(Windows 环境推荐)

### 本指南以 Windows 系统的 CMD 终端为例，手把手带你完成提取。

\### 1. 下载项目到本地

打开命令提示符（快捷键 `Win + R`，输入 `cmd`），将本项目克隆到你的电脑上：

```cmd

git clone \[https://github.com/yiyunwei10-oss/extract-getnote-spa-version.git](https://github.com/yiyunwei10-oss/extract-getnote-spa-version.git)

继续输入以下命令并回车：

npx playwright install



## 🚀使用方法

### 第一步：订阅知识库

**什么是 Get笔记知识库？**

Get笔记提供了"知识库"功能，可以订阅抖音博主的短视频文案。订阅后，博主发布的视频文案会自动同步到你的知识库中。

**如何订阅？**

1. **打开 Get笔记 APP（可在应用商店搜索下载）**
2. **在首页点击“知识库”标签，或在抖音中复制博主主页链接后在 Get笔记中搜索。**
3. **进入博主的知识库页面，点击“订阅”或“关注”。**

### 第二步：获取知识库URL

打开博主的知识库页面，复制完整的浏览器地址：

```
https://www.biji.com/subject/QYARpjM0/DEFAULT?followId=785142\\\\\\\&followName=博主名称
```

⚠️ **重要**：必须复制完整URL，包含 `followName` 参数，这样才能正确命名输出文件夹。

### 第三步：运行提取命令

**在终端执行 node run.js 命令，并在后面用双引号括起你需要提取的完整知识库 URL：**

```bash
node run.js "https://www.biji.com/subject/QYARpjM0/DEFAULT?followId=..."
```

### 第四步：扫码登录与全自动抓取

命令执行后，会弹出一个浏览器窗口，并为你预留 3 分钟 的安全等待时间。



请在弹出的浏览器窗口中，完成 Get笔记的扫码或账号登录操作。



登录成功并加载出左侧文章列表后，无需多余操作，脚本会自动接管，开始“点击列表 -> 跳转新页 -> 提取全文 -> 后退”的自动化循环。



## 📁 输出格式

提取的文章会保存在当前目录下，以 followName 参数命名的文件夹中：

/extract-getnote-spa-version/

├── 博主A/              # 第一次提取

│   ├── 001\_文章标题.md

│   ├── 002\_文章标题.md

│   └── ...

├── 博主A\_2/            # 第二次提取（若出现重名，会自动添加序号避免冲突）

│   ├── 001\_文章标题.md

│   └── ...



每篇文章包含：

* 文章标题
* 原始链接
* 完整正文内容

## 常见问题

**Q: 如何停止提取？**

A: 按 `Ctrl+C`，脚本会等待当前批次完成后安全退出。下次运行会自动继续。

**Q: 提取到一半中断了怎么办？**

A: 重新运行相同的命令，脚本会自动跳过已提取的文章，从中断处继续。

**Q: 为什么文件夹名后面有 \_2、\_3？**

A: 这是为了避免重名冲突。如果你多次提取同一个博主，会自动添加序号。

**Q: 如何提高提取速度？**

A: 可以增加并发数，但建议不超过5，避免被限流。

**Q: 提取的内容不完整？**

A: 确保已在 Get笔记 APP 中订阅该知识库，并且浏览器已登录。

## 关于合法性

### 这个工具是合法的

本工具通过浏览器自动化访问你已订阅的 Get笔记内容。这是合法的，原因如下：

**1. 授权访问**

* 你使用自己的账号和订阅权限
* 只能访问你已订阅的知识库
* 工具不会绕过任何技术保护措施或访问限制

**2. 浏览器自动化 ≠ 网络爬虫**

* 工具使用 Playwright 自动化浏览器操作
* 相当于你手动点击、浏览、复制的自动化版本
* 类似于浏览器插件、自动填表工具等合法工具

**3. 内容来源**

* Get笔记已经合法收集了这些内容
* 你通过订阅获得了查看权限
* 工具只是帮你整理你有权访问的内容

**法律类比：**
这就像你把 Netflix 上看过的电影列表导出来做笔记，或者用工具整理你的 Notion 笔记。你有权限访问的内容，用工具帮你整理，完全合法。
